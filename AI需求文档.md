# 双色球开奖与 AI 预测展示网页需求文档

## 1. 项目概述

### 1.1 项目名称
双色球开奖与 AI 预测数据展示系统

### 1.2 项目目标
开发一个单页面 Web 应用，用于展示双色球历史开奖数据以及多个 AI 模型的预测结果。数据通过外部 JSON 文件加载，便于手动更新维护。

### 1.3 技术栈建议
- **前端框架**: Vue.js 3 / React / 纯 HTML+CSS+JavaScript（任选其一）
- **样式**: Tailwind CSS / Bootstrap / 自定义 CSS
- **数据格式**: JSON
- **部署**: 静态网页，可部署到任何 Web 服务器或 GitHub Pages

---

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 历史开奖数据展示
- 以表格形式展示双色球历史开奖记录
- 显示字段：期号、开奖日期、红球（6个）、蓝球（1个）
- 支持按期号降序排列（最新期在最上方）
- 红球和蓝球需要有视觉区分（颜色标识）
- 支持分页或虚拟滚动，避免一次性加载过多数据

#### 2.1.2 AI 预测数据展示
- 展示多个 AI 模型的预测结果
- 每个 AI 模型显示：
  - 模型名称
  - 预测的 5 组号码（每组包含 6 个红球 + 1 个蓝球）
  - 预测策略说明（可选）
- 号码展示需要有清晰的视觉效果（球形图标或彩色标签）

#### 2.1.3 数据对比功能
- 可选功能：将 AI 预测号码与最新一期开奖结果进行对比
- 高亮显示预测命中的号码
- 显示每组预测的命中数量（红球命中数 + 蓝球是否命中）

### 2.2 用户交互

#### 2.2.1 数据筛选
- 可按日期范围筛选历史开奖数据
- 可选择显示特定 AI 模型的预测

#### 2.2.2 数据刷新
- 提供"刷新数据"按钮，重新加载 JSON 文件
- 显示数据最后更新时间

#### 2.2.3 响应式设计
- 支持桌面端、平板和移动端访问
- 在小屏幕设备上优化表格显示（横向滚动或卡片式布局）

## 3. 数据结构定义

### 3.1 历史开奖数据文件 (`lottery_history.json`)

```json
{
  "last_updated": "2025-10-21T10:00:00Z",
  "data": [
    {
      "period": "25120",
      "date": "2025-10-19",
      "red_balls": ["01", "02", "04", "07", "13", "32"],
      "blue_ball": "07"
    },
    {
      "period": "25119",
      "date": "2025-10-16",
      "red_balls": ["06", "09", "23", "26", "28", "32"],
      "blue_ball": "11"
    }
  ]
}
```

**字段说明：**
- `last_updated`: 数据最后更新时间（ISO 8601 格式）
- `data`: 开奖记录数组
  - `period`: 期号（字符串）
  - `date`: 开奖日期（YYYY-MM-DD 格式）
  - `red_balls`: 红球号码数组（6个元素，每个为两位数字字符串）
  - `blue_ball`: 蓝球号码（两位数字字符串）

### 6.1 数据加载
```javascript
// 示例代码（伪代码）
async function loadLotteryData() {
  const response = await fetch('./lottery_history.json');
  const data = await response.json();
  return data;
}

async function loadPredictions() {
  const response = await fetch('./ai_predictions.json');
  const data = await response.json();
  return data;
}
```

### 6.2 号码对比逻辑
```javascript
// 示例代码（伪代码）
function compareNumbers(prediction, actualResult) {
  const redHits = prediction.red_balls.filter(ball => 
    actualResult.red_balls.includes(ball)
  ).length;
  
  const blueHit = prediction.blue_ball === actualResult.blue_ball;
  
  return {
    red_hits: redHits,
    blue_hit: blueHit,
    total_hits: redHits + (blueHit ? 1 : 0)
  };
}
```

### 6.3 响应式断点
- 桌面端：≥ 1024px
- 平板：768px - 1023px
- 移动端：< 768px
